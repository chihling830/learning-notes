# CS50 Week 0 影片內容整理與摘要

這段影片是**哈佛大學知名計算機科學入門課程 CS50 的第一堂課 (Week 0)**，由大衛·馬蘭 (David Malan) 教授授課。您提供的時間戳記 (3016秒，約 50分16秒) 剛好落在課程的後半段 —— 教授正透過生活化的例子與遊戲實作，講解演算法原理及示範 Scratch 程式設計。

以下為影片的完整核心重點整理：

### 1. AI 時代的程式設計與學習
*   **AI 成為強大助手**：影片開頭提到 AI (如 ChatGPT, GitHub Copilot 等) 將改變寫程式的方式。未來的程式設計師更像是「指揮家」或「飛行員」，懂底層原理後，能快速利用 AI 幫忙寫程式、找 bug，大幅提升效率。
*   **實作示範**：教授現場示範用 Python 和 OpenAI API 寫出一個簡單的 ChatBot，僅用十行程式碼就做出能改變個性的 AI (例如：叫它表現得像隻貓)。
*   **黃色小鴨除錯法 (Rubber Duck Debugging)**：引進了程式設計界知名的除錯方式，並介紹了 CS50 開發的 AI 助教 (CS50.AI 虛擬小鴨) 來幫助學生學習。

### 2. 電腦科學的底層邏輯：全都是 0 與 1
*   **硬體基礎 (二進位制)**：電腦的底層只有「通電 (1)」和「不通電 (0)」的電晶體開關。透過**二進位 (Binary System)**，用很少的開關組合就能表示不同的數字。
    *   一個開關叫一個 **Bit (位元)**，8個 Bits 組成一個 **Byte (位元組)**。
*   **文字的表示 (ASCII 與 Unicode)**：
    *   用特定的數字組合來代表英文字母 (例如：數字 65 代表大寫 'A')，這叫做 ASCII 編碼。
    *   為了容納全世界的語言甚至 Emoji (表情符號)，後來延伸發展出能用更多 Bits 表示文字的 **Unicode** 標準。
*   **顏色、圖像與多媒體的表示**：
    *   **顏色 (RGB)**：用三個數字 (紅、綠、藍光的強度參數：0~255) 混合出所有顏色。
    *   **圖片**：由無數個有顏色的點 (Pixels) 拼湊而成。
    *   **影片**：以每秒數十張圖片快速連續播放產生的視覺錯覺。
    *   **聲音**：用數字記錄音調 (頻率)、音量 (振幅) 和持續時間而組成。

### 3. 解題的藝術：演算法 (Algorithms)
教授透過「尋找電話簿裡的 John Harvard」這個生動的例子，說明了演算法 (解決問題的步驟) 的重要性：
*   **線性搜尋法 (一頁一頁找)**：一定找得到，但是效率最慢 (Linear, 取決於頁數 N)。
*   **兩頁一找**：速度快一倍，但可能會漏掉。
*   **二元搜尋法 (Binary Search, 從中間把電話簿撕一半)**：這就是電腦與資料庫的高效搜尋方式 (Logarithmic)。即便電話簿變成兩倍厚，也只需要多撕一次就好，速度極快極有效率。

### 4. 虛擬碼與程式設計的核心概念
在進入真實的程式碼前，教授介紹了幾個放諸四海皆準的程式邏輯基礎：
*   **函式 (Functions)**：執行特定動作的動詞 (例如：印出文字、發出聲音)。
*   **條件式 (Conditionals/Boolean Expressions)**：像是十字路口，透過詢問「是或否」(例如：碰到了牆壁嗎？) 來決定接下來要做什麼。
*   **迴圈 (Loops)**：讓電腦重複執行某些步驟，不再需要手動重寫超多行相同的程式碼。

### 5. 視覺化程式語言：Scratch 實作 (影片後半段重點)
為了抽象化枯燥的語法 (如括號、分號)，教授帶領學生使用 MIT 開發的 **Scratch** 視覺化程式工具來體驗寫程式：
*   **打造第一支程式**：將方塊拼疊起來，完成輸入名字並讓貓咪向你打招呼的功能。
*   **奧斯卡的接垃圾遊戲 (Oscar Time)**：示範如何拆解複雜遊戲的邏輯——垃圾是由上往下的無限輪迴降落，而奧斯卡則是偵測滑鼠事件改變樣貌。
*   **哈佛大戰長春藤 (Ivy's Hardest Game)**：由前幾屆學生寫的彈幕閃避遊戲。示範了如何透過 X/Y 座標移動、邊界偵測 (碰到牆壁反彈) 以及敵人自動追蹤的演算法 (如永遠面向主角前進 10 步) 實作出高難度的遊戲邏輯。

這部影片是寫給完全新手的電腦科學入門指南，從最底層的二進位硬體邏輯、演算法效率，一路引導到使用有趣圖形化介面寫出你的第一個遊戲。無論是人工智慧、硬體邏輯還是寫遊戲，本質上都是在「**輸入資料 $\rightarrow$ 透過演算法處理 $\rightarrow$ 輸出結果**」。
